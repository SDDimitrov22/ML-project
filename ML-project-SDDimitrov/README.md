# 🤖 AI Shopping Assistant (RAG System)

Този проект представлява интелигентен асистент за онлайн магазин, базиран на Retrieval-Augmented Generation (RAG) технология. Системата съчетава векторна база данни с генеративен изкуствен интелект за обслужване на клиенти на български език.

---

## 📁 Структура на проекта
* ml_project.ipynb — Основен Jupyter Notebook с изходния код и демонстрация.
* requirements.txt — Списък с необходимите Python библиотеки.
* .env — Конфигурационен файл за API ключове.
* README.md — Инструкции и подробен отчет по проекта (този файл).

---

## 🛠️ Технически стек
* Weaviate Cloud: Векторна база данни за семантично търсене.
* Cohere API: За векторизация (Embeddings) и генериране на отговори.
* Python 3.12+: Език за разработка.

---

## ⚙️ Инсталация и стартиране

1. Инсталирайте библиотеките:
   Отворете терминала в PyCharm и изпълнете:
   pip install -r requirements.txt

2. Настройте API ключовете:
   Уверете се, че в папката има файл .env със следните променливи:
   - WEAVIATE_URL
   - WEAVIATE_API_KEY
   - COHERE_API_KEY

3. Стартиране:
   Отворете ml_project.ipynb и изпълнете всички клетки последователно (Run All).

## 📝 Кратък отчет по проекта

### 1. Цел на проекта
Целта е разработването на интелигентна система, която помага на клиентите да намерят продукти чрез естествен език. Вместо стандартно търсене по ключови думи, системата разбира смисъла на въпроса (напр. "гейминг", "музика", "бюджетни решения").

### 2. Архитектура на системата
Системата следва класическа RAG архитектура:
* Retrieval (Извличане): Потребителската заявка се векторизира и се изпраща към Weaviate Cloud. Базата данни връща най-релевантните продукти чрез векторен модел.
* Augmentation (Обогатяване): Намерените данни (име и цена) се вмъкват в промпт, който се изпраща към езиковия модел.
* Generation (Генерация): LLM (Cohere) генерира любезен отговор на български език, базиран единствено на извлечените данни.

### 3. Описание на колекциите и данните
В Weaviate е дефинирана колекция "Product" със следните свойства:
* name (Text): Име на продукта. Използва се за семантично търсене.
* price (Number): Цена на продукта. Използва се за логически филтри.

Примерен набор от данни в базата:
* Laptop X1 (1200.0 лв.)
* Wireless Headphones (150.0 лв.)
* Gaming Mouse (50.0 лв.)

### 4. Използвани агенти
Проектът използва концепцията за Generative Search Agent. Агентът действа като виртуален консултант, който е инструктиран да:
* Комуникира само на български език.
* Предоставя информация само на база реалния каталог.
* Прави интелигентни сравнения (напр. кой е най-евтиният продукт).

### 5. Алгоритми за обработка
* Cosine Similarity: Алгоритъмът в Weaviate, който изчислява ъгъла между вектора на въпроса и векторите на продуктите в многомерното пространство.
* Vectorization (Embeddings): Моделът text2vec-cohere трансформира думите в числови вектори от 1024+ измерения.
* Prompt Engineering: Техника за структуриране на входните данни към AI модела, за да се гарантира точност и правилен тон на български език.

### 6. Примерни входове и изходи
* Вход: "Имате ли нещо за геймъри?"
* Изход: "За геймъри предлагаме Gaming Mouse (50 лв.) и Laptop X1 (1200 лв.)."
* Вход: "Предложи ми продукт под 100 лева."
* Изход: "Най-подходящият продукт в този бюджет е Gaming Mouse на цена 50.0 лв."

### 7. Ограничения и идеи за бъдещо развитие
* Ограничения: Зависимост от външни облачни API; липса на памет за предходни съобщения в реално време (Session Memory).
* Бъдещо развитие: Добавяне на история на чата, интеграция с реална количка за пазаруване и поддръжка на гласови команди.
